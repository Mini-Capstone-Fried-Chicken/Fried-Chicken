# Security & API Keys Configuration

## Overview
This project uses sensitive API keys and credentials. To maintain security, please follow these guidelines.

## Setup Instructions

### 1. Environment Variables

The project requires the following environment variables to be set:

```bash
# Google Maps Directions API key
GOOGLE_DIRECTIONS_API_KEY=your_key_here

# Google Maps Places API key
GOOGLE_PLACES_API_KEY=your_key_here
```

### 2. Local Development Setup

**Step 1:** Copy the example file
```bash
cp .env.example .env
```

**Step 2:** Edit `.env` and add your actual API keys
```
GOOGLE_DIRECTIONS_API_KEY=AIzaSy...
GOOGLE_PLACES_API_KEY=AIzaSy...
```

**Step 3:** Run using the helper scripts (recommended)

**On Linux/macOS:**
```bash
chmod +x flutter-run.sh
./flutter-run.sh run              # Run the app
./flutter-run.sh test             # Run tests
```

**On Windows:**
```bash
flutter-run.bat run               # Run the app
flutter-run.bat test              # Run tests
```

**Alternative:** Run manually with environment variables
```bash
flutter run --dart-define=GOOGLE_DIRECTIONS_API_KEY=$(cat .env | grep GOOGLE_DIRECTIONS_API_KEY | cut -d '=' -f2)
flutter test --dart-define=GOOGLE_DIRECTIONS_API_KEY=$(cat .env | grep GOOGLE_DIRECTIONS_API_KEY | cut -d '=' -f2)
```

### 3. CI/CD Setup (GitHub Actions, etc.)

Use secrets management in your CI/CD platform:

1. **GitHub:** Add repository secrets
   - Go to: Settings → Secrets and variables → Actions → New repository secret
   - Add `GOOGLE_DIRECTIONS_API_KEY`
   - Add `GOOGLE_PLACES_API_KEY`

2. **In your workflow file** (e.g., `.github/workflows/test.yml`):
   ```yaml
   - name: Run tests
     run: |
       flutter test \
         --dart-define=GOOGLE_DIRECTIONS_API_KEY=${{ secrets.GOOGLE_DIRECTIONS_API_KEY }} \
         --dart-define=GOOGLE_PLACES_API_KEY=${{ secrets.GOOGLE_PLACES_API_KEY }}
   ```

3. **For other platforms** (GitLab, CircleCI, etc.):
   - Refer to their secret management documentation
   - Set environment variables before running Flutter commands

### 4. Firebase Configuration
Firebase options are stored in `lib/services/firebase/firebase_options.dart`. These are generated by FlutterFire CLI and are generally safe to commit as they contain:
- Project IDs (public)
- API Keys with platform restrictions (can be public)

However, ensure your Firebase project has proper security rules configured in:
- Cloud Firestore Security Rules
- Realtime Database Rules
- Cloud Storage Security Rules

### 5. Revoking Compromised Keys

If a key is accidentally committed or exposed:

1. **Immediately revoke** the key in Google Cloud Console:
   - Go to [https://console.cloud.google.com/](https://console.cloud.google.com/)
   - Navigate to APIs & Services → Credentials
   - Find and delete the exposed key

2. **Generate a new key** with proper restrictions:
   - Application restriction: Android, iOS, Web
   - API restriction: Directions API and Places API

3. **Update your `.env` file** with the new key

4. **Check git history** for previous commits:
   ```bash
   git log --all --full-history -- "lib/services/google_directions_service.dart"
   ```

5. **Consider force-pushing** if the repository is private and recent, or using git-filter-repo for larger repositories

### 6. Best Practices

✅ **DO:**
- Store API keys in environment variables
- Use `.env.local` for development
- Add secrets to CI/CD platform
- Restrict API keys to specific APIs and platforms
- Regularly rotate keys
- Monitor API usage in Google Cloud Console

❌ **DON'T:**
- Commit `.env` files to git
- Share API keys in chat, email, or code reviews
- Use the same key across development/production
- Give keys unnecessary permissions
- Leave keys in public repositories
